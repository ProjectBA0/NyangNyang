# Changelog

## [2025-12-19] - 프론트엔드 레이아웃 구축 및 UI 시스템 정교화

### 1. 시스템 레이아웃 규격 확립 (시행착오 및 개선 플로우)
*   **초기 단계**: 1600px 기반의 기본 레이아웃 구성 및 개별 페이지(Event, Main) 생성.
*   **중간 단계**: 실제 시각적 가시성 확보를 위해 1200px 콘텐츠 영역 설정 및 전역 Navbar 리팩토링.
*   **최종 단계**: 전체 배경 1920px, 콘텐츠 핵심 영역 1200px 중앙 정렬 규격 확정 및 모든 컴포넌트 적용.

### 2. UI 컴포넌트 고도화 및 공통화
*   **Navbar & Footer 통합**: 페이지별로 파편화되어 있던 헤더와 푸터를 공통 컴포넌트(`components/`)로 분리하고 `MainLayout`에 적용하여 전역 일관성 확보.
*   **MainPage 재구성**: 인라인 스타일을 제거하고 `App.css` 기반의 클래스 시스템으로 전면 전환.
*   **Footer 레이아웃 특수화**: 푸터 배경을 전체가 아닌 중앙 1200px 영역에만 검은색으로 적용하여 세련된 여백 처리 구현.

### `front/src/components/Footer.jsx`
*   소셜 섹션에 Instagram, YouTube, Naver Blog 아이콘(이모지 플레이스홀더) 추가.
*   고객센터 운영시간을 한 줄로 표시하도록 레이아웃 수정 (`whiteSpace: nowrap` 적용).

### `front/src/pages/CustomerCenterPage.jsx` (새 파일)
*   고객 문의 및 공지사항을 위한 고객센터 페이지 구축.
*   InfoBox, CategoryBar, Board 구성을 통한 정보 전달 레이아웃 구현.
*   전역 `Navbar`, `Footer` 통합 및 `App.css` 스타일 시스템 적용.

### `front/src/App.css` (색상 재배치 및 디버그 스타일 업데이트)
*   전체 페이지 너비를 1920px로 확장 (`--page-width: 1920px`).
*   고객센터 페이지 전용 레이아웃 변수 및 섹션별 고유 배경색 추가.
*   모든 섹션에 겹치지 않는 고유 배경색 부여:
    *   헤더: 빨강 계열 / 로고: 초록 / 검색: 파랑 / 우측: 노랑 / 내비: 보라
    *   메인: 슬라이더(청록), 카테고리(갈색), 상품(남색), 하단(올리브/자주)
    *   고객센터: 안내박스(하늘색), 카테고리바(노랑), 검색바(연두), 게시판(회색/보라)
*   푸터 배경을 중앙 1200px만 검은색으로 설정하고 양옆을 여백으로 처리함.
*   디버그 라벨 색상을 하늘색(`rgba(0, 255, 255, 0.3)`)으로 통일하고 투명도 조정.
*   파일 최하단에 상세 변경 이력 주석 추가.

### 3. 레이아웃 검증 시스템 (Debug Styles) 개선 과정
*   **가독성 확보**: 초기 점선(Dashed) 테두리에서 발생하던 시각적 혼란을 제거하고 반투명 배경색 시스템으로 전환.
*   **겹침 문제 해결**: 
    *   디버그 라벨 위치를 `top-left`에서 `bottom-right`로 이동하여 콘텐츠와 분리.
    *   `Global Header`와 `Grid Section` 등 컨테이너 라벨은 예외적으로 `top-left`에 배치하여 내부 요소와 겹침 방지.
*   **시각적 구분**: 모든 섹션에 겹치지 않는 고유 색상을 부여하고, 헤더 영역은 하늘색(`Cyan`)으로 특수 지정하여 가시성 극대화.
